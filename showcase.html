Best Practices & Common Mistakes

Learn from common mistakes and follow best practices to ensure your code is consistent, accessible, and maintainable.

Do's and Don'ts

✅ Do's

  • Use utility classes for spacing: .mt-4, .py-6
  • Use CSS custom properties: var(--color-primary)
  • Include ARIA labels on interactive elements
  • Test components in both light and dark themes
  • Use semantic HTML5 elements
  • Follow BEM naming convention
  • Use safeQuery / safeQueryAll from utils.js
  • Add descriptive alt text to images

❌ Don'ts

  • Don't use inline styles: style="padding: 20px"
  • Don't hardcode colors: color: #000000
  • Don't skip ARIA labels on icon-only buttons
  • Don't use duplicate IDs across pages
  • Don't use document.querySelector directly
  • Don't forget to test in both themes
  • Don't use non-semantic elements for structure
  • Don't forget alt text on images

Common Mistakes to Avoid

1. Inline Styles Instead of Utility Classes

❌ Wrong:

<div style="padding: 20px; margin-top: 16px;">
    Content
</div>

✅ Correct:

<div class="py-4 mt-2">
    Content
</div>

Why: Utility classes are theme-aware, maintainable, and consistent with the design system.

2. Hardcoded Colors

❌ Wrong:

.my-element {
    color: #000000;
    background: #ffffff;
}

✅ Correct:

.my-element {
    color: var(--color-text);
    background: var(--color-bg);
}

Why: CSS custom properties automatically adapt to light/dark themes and can be changed globally.

3. Missing ARIA Labels

❌ Wrong:

<button class="theme-toggle">
    <img src="sun.svg" alt="Sun">
</button>

✅ Correct:

<button class="theme-toggle" 
        type="button" 
        aria-label="Toggle color theme">
    <img src="sun.svg" alt="Light theme">
</button>

Why: ARIA labels make components accessible to screen readers and assistive technologies.

4. Direct DOM Manipulation

❌ Wrong:

const element = document.querySelector('.my-class');
element.classList.add('active');

✅ Correct:

const element = safeQuery('.my-class');
if (element) {
    element.classList.add('active');
}

Why: safeQuery handles null checks and prevents runtime errors.

5. Not Testing in Both Themes

Common Issues:

  • Text becomes unreadable in dark theme
  • Borders disappear or become too prominent
  • Shadows don't work well in both themes
  • Images with transparency look wrong

Solution: Always use theme-aware CSS variables and test with the theme toggle.

Accessibility & Touch Optimization

Accessibility Features

  • Focus States: All interactive elements have visible focus indicators using focus-visible with 3px outline and 2px offset.
  • Keyboard Navigation: Full keyboard support for all components. Modals trap focus and can be closed with Esc.
  • ARIA Support: Modals use aria-modal and role="dialog". Accordions use aria-expanded. Tabs use proper ARIA roles: tablist, tab, tabpanel.
  • Color Contrast: All text meets WCAG AA standards (4.5:1 for normal text, 3:1 for large text).
  • Semantic HTML: Proper heading hierarchy, landmark regions, and semantic elements throughout.
  • Screen Reader Support: Tooltips visible on focus, proper labels for form controls, descriptive alt text patterns.

Touch Device Optimization

  • Touch Targets: All interactive elements meet WCAG 2.5.5 minimum size of 44×44px (comfortable 48px).
  • Spacing: Increased spacing between touch targets (minimum 8px) to prevent accidental taps.
  • Tap Highlight: Removed default iOS tap highlight for cleaner interactions.
  • Touch Scrolling: Smooth scrolling with -webkit-overflow-scrolling: touch for better mobile performance.
  • Form Controls: Checkboxes and radio buttons enlarged to 20×20px on touch devices.
  • Active States: Visual feedback on touch with scale transforms and background changes.

Interactive States

All interactive components include comprehensive state management:

Button States

  • :hover - Subtle lift with shadow
  • :active - Pressed state (scale down)
  • :focus-visible - Keyboard focus ring
  • :disabled - Reduced opacity, no pointer events

Form States

  • :focus - Border color change + focus ring
  • :disabled - Grayed out, not interactive
  • :invalid - Error styling (when implemented)
Example: Button with all states

  transition: transform 0.2s ease;
}
