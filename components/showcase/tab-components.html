Best Practices & Common Mistakes

Learn from common mistakes and follow best practices to ensure your code is consistent, accessible, and maintainable.

Do's and Don'ts

‚úÖ Do's

  • Use utility classes for spacing: .mt-4, .py-6
  • Use CSS custom properties: var(--color-primary)
  • Include ARIA labels on interactive elements
  • Test components in both light and dark themes
  • Use semantic HTML5 elements
  • Follow BEM naming convention
  • Use safeQuery / safeQueryAll from utils.js
  • Add descriptive alt text to images

‚ùå Don'ts

  • Don't use inline styles: style="padding: 20px"
  • Don't hardcode colors: color: #000000
  • Don't skip ARIA labels on icon-only buttons
  • Don't use duplicate IDs across pages
  • Don't use document.querySelector directly
  • Don't forget to test in both themes
  • Don't use non-semantic elements for structure
  • Don't forget alt text on images

Common Mistakes to Avoid

1. Inline Styles Instead of Utility Classes

‚ùå Wrong:

<div style="padding: 20px; margin-top: 16px;">
    Content
</div>

‚úÖ Correct:

<div class="py-4 mt-2">
    Content
</div>

Why: Utility classes are theme-aware, maintainable, and consistent with the design system.

2. Hardcoded Colors

‚ùå Wrong:

.my-element {
    color: #000000;
    background: #ffffff;
}

‚úÖ Correct:

.my-element {
    color: var(--color-text);
    background: var(--color-bg);
}

Why: CSS custom properties automatically adapt to light/dark themes and can be changed globally.

3. Missing ARIA Labels

‚ùå Wrong:

<button class="theme-toggle">
    <img src="sun.svg" alt="Sun">
</button>

‚úÖ Correct:

<button class="theme-toggle" 
        type="button" 
        aria-label="Toggle color theme">
    <img src="sun.svg" alt="Light theme">
</button>

Why: ARIA labels make components accessible to screen readers and assistive technologies.

4. Direct DOM Manipulation

‚ùå Wrong:

const element = document.querySelector('.my-class');
element.classList.add('active');

‚úÖ Correct:

const element = safeQuery('.my-class');
if (element) {
    element.classList.add('active');
}

Why: safeQuery handles null checks and prevents runtime errors.

5. Not Testing in Both Themes

Common Issues:

  • Text becomes unreadable in dark theme
  • Borders disappear or become too prominent
  • Shadows don't work well in both themes
  • Images with transparency look wrong

Solution: Always use theme-aware CSS variables and test with the theme toggle.
